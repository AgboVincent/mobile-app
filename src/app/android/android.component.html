PROJECT ONE

DRIVER TRAVEL REQUEST MOBILE APP

The functionality of the android application is to discretely assign available and competent drivers already registered on the app data base to end users who own a car but not willing to drive at the moment

OBJECTIVE

To design a user friendly interface for both end user and driver application.
To design and program a system that enable viewing of available registered driver and there location.
To program a system that automatic bills the end user.

SCOPE

This work cover wide range of mobile App features which include registration (Authentication), the login, interface, driver timing, rider location and destination, end user feedback and the automating billing in accordance with parameters considered.

THE DRIVER APP

REGISTRATION AND SIGN IN

We designed the register and the login view in the MainActivity.java and in the mainactivity.xml which is the first page the Driver will see when he clicks on the app. The background picture was designed and we also added some directive text to guide the Driver.

Register dialogue was created and the following features was added as details of the driver while registering

NAME: This feature holds the full name of the driver.

EMAIL: This feature requires a functional email of the driver.

AGE: This holds the real age of the driver.

PHONE: The driver’s active phone number is inserted here.

PASSWORD: This requires a strong password that the driver can remember always.

WORK EXPERIENCE: This require the year(s) experience the driver has gathered in his driving time.

UPLOAD PASSPORT: The driver is required to upload a clear passport photograph in this field.

UPLOAD DRIVERS LICENSE: The driver uploads his personal valid driver’s license.

After the driver has added all the details, he click the submit button. A popup message will be displayed indicating whether it was successfully registered or not. The details will be stored in the firebase database if it was successfully registered. When the driver has successfully registered, he can proceed by clicking the SIGN IN BUTTON. The driver is also required to sign in with email and password that was used to register. An error message will be displayed if the text entered in the fields didn’t match with the one used in registration.

RIDER/USER APP

REGISTER AND SIGN IN

The rider app project was created and added to the same database as the driver app in firebase database. The layout for the register and login was setup. Every detail of the rider is stored in same database as the driver

SHOW EXACT LOCATION

Google map was added to the rider app to make it show the exact place of the rider when the rider log’s in. location services was also integrated from google play and geo fire API was enabled in the rider app.

PICKUP REQUEST

We created functions for the pickup request of the rider by the tap of pickup Request button. Pickup location and destination was designed, calculate distance and time method was used to get fare estimate. I added all the library that was expected.

CALCULATE BILL OF SERVICE

A dialogue for display location, destination and estimated billing of the user was created. It was created through the custom marker info. This dialogue also displayed the estimated distance and time for the travel.

LOAD ALL AVAILABLE DRIVERS

The find driver section was designed. Functions to load all available drivers with the range of 5 km was developed. The rider can view available drivers from the app. The database info for this function was equally updated.

INFORMATION STORED IN THE DATABASE WITH THEIR REFERENCE NAME

Drivers: Table store information about Location of Driver that are available

Pickup Request: Table store information about pickup Request of Users

Riders Information: Store information of users who register from the Rider App

Drivers Information: This table store information about Driver who register from Driver app

FIREBASE MESSAGING

Firebase messaging library and retrofit and Gson converter was added. We also added Refreshed Token to the app, Token table was created in the database. Whenever a new request is made, a new token for the request is generated.

PRESENCE SYSTEM

Presence system was integrated into the Driver app and Rider App. The driver can be able to turn off his presence when he is busy and turn on when his is not busy. The rider will not see any driver available when the driver is offline. We updated real time available drivers in the driver app

DRIVER NOTIFICATION

The driver gets a notification as a call when the user makes a request by the rider. We created alert driver method. A request notification was sent to the driver on request. The driver gets a call notification when a request is put to him by a rider. We used advanced rest client software (ARC) to generate a http post notification to the driver when request are made

ACCEPT AND DECLINE BUTTON

ACCEPT AND DECLINE buttons was added on the driver app to enable him accept or decline bookings put to them by the riders. When the accept button is clicked it will show direction from driver location to customer real time

DRIVER ARRIVAL MESSAGE

An additional function was added to send a message to the rider when the driver arrived the pickup location.
